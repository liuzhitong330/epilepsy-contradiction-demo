EpiContradiction

An automated pipeline that reads epilepsy research papers and surfaces candidate contradictions to define intervention hypotheses, using LLM-based claim extraction and rule-based contradiction detection.

Background

The problem

Epilepsy research spans thousands of papers across different model organisms, cell types, and disease subtypes. A finding in mouse granule cells may directly contradict a finding in human TSC tissue — but no human researcher reads everything.

Contradictions are not just inconsistencies — they define where biological systems are under-specified and where intervention strategies are most uncertain. They are nearly impossible to surface manually at scale.

EpiContradiction automates this process: structured claim extraction → pairwise contradiction detection → ranked output with experiment suggestions.

System Design

How it works

Four automated stages from raw literature search to ranked, verified contradictions.

1
Retrieval
Literature Retrieval
PubMed Entrez API · Biopython · Rate-limited (3 req/s) · Parallel threads
Input7 domain-specific search queries · last 10 years · abstracts ≥ 150 words
Output287 deduplicated papers tagged with matched search terms (papers.json)
2
Extraction
Claim Extraction
Claude Sonnet · JSON schema enforcement · Batched (10/batch) · Resumable
InputAbstract text per paper with structured system prompt
Output1634 mechanistic claims — each with subject / direction / effect / condition / confidence (claims.json)
3
Detection
Contradiction Detection
2-stage: rule-based pre-filter + LLM judge
3a — Rule engine: subject fuzzy match (substring) + opposing direction pairs (increases↔decreases, activates↔inhibits) + effect word overlap filter. Deduplicates by PMID pair, keeps highest-scoring claim match.
3b — LLM judge: each candidate pair classified as true_contradiction / conditional_contradiction / terminology_mismatch / not_contradictory with explanation and resolving experiment suggestion.
Input1,634 × 1,634 claim comparisons → 3 candidates after filters
Output1 adjudicated contradiction candidate (contradictions.json)
4
Output
Ranked Output
Interest scoring · match score + LLM confidence + experiment feasibility

Research agenda

From contradictions to intervention optimization

A research agenda

1
Detection
Contradiction detection

The pipeline surfaces candidate contradictions across studies, identifying where the effect of a mechanism or intervention is inconsistent across experimental contexts. These are not noise — they are signals of under-specified biology.

2
Hypothesis
Hypothesis space

Each contradiction implies one or more hidden variables that modulate the observed effect:

disease subtype circuit state experimental context

Resolving which variable is responsible narrows the hypothesis space for targeted intervention design.

3
Intervention
Intervention question

Rather than resolving contradictions analytically, we reframe them as optimization problems: what intervention parameters produce consistent, context-robust seizure suppression across experimental conditions?

4
Future
Closed-loop optimization future direction

This reframing enables a closed-loop system in which contradiction detection feeds directly into computational optimization:

LLM proposes parameters → Digital twin simulates response → RL evaluates suppression → Parameters refined

e.g. drug dosage, stimulation frequency, or circuit targeting — evaluated against a neural simulation such as The Virtual Brain (TVB)

EpiContradiction serves as a problem generator — identifying high-value regions of the literature where intervention optimization is most needed, and where the cost of under-specification is highest.

Principles

Design philosophy

This system is built around inspectable reasoning. Every output — extracted claim, candidate pair, classification decision — can be traced back to the source abstract and the exact logic that produced it.

The pipeline deliberately separates rule-based filtering from LLM judgment, so failure modes are locatable: if a candidate pair is wrong, it is either a fuzzy match error (Stage 3a) or an LLM classification error (Stage 3b), not an inscrutable black box.

Honest limitations are documented inline rather than buried in supplementary material. The goal is a system a domain expert can audit, not just a result they are asked to trust.

Honest assessment

Limitations & future work

What this system does well and where it falls short.

Current limitations

  • ⚠️Abstract-only analysis — full text not accessed, missing methods sections and results tables
  • ⚠️2025–2026 publication window only — insufficient time for contradictions to accumulate in the literature
  • ⚠️Fuzzy substring matching misses synonyms (Nav1.1 vs SCN1A, GABA-A vs GABA receptor)
  • ⚠️No embedding-based semantic similarity — entity resolution is purely lexical
  • ⚠️Single domain (epilepsy only) — cross-disease signal not captured

Planned extensions

  • →Full-text PDF parsing via PubMed Central Open Access API
  • →Embedding-based entity resolution with BioBERT for synonym handling
  • →Expand to 10-year window for richer contradiction mining
  • →Cross-disease contradiction detection (epilepsy ↔ Alzheimer's, ASD)
  • →Knowledge graph visualization of the full claim network
